1. Client Request

A user (client) enters a URL in the browser, like /ex01/display. This triggers an HTTP request, which is sent to the Django server.

2. URL Routing

Django's URL dispatcher looks for a matching URL pattern in the urls.py file. In this case, it will find the route path('ex01/display', views.display, name='display').

If a match is found, Django forwards the request to the corresponding view function (display in this example).

3. View Function

The view function in views.py handles the request. It typically fetches or processes data (if necessary) and selects a template to render.

For a static page, the view is simple. It loads the template and passes any necessary context data (in this case, there might be no dynamic data).

from django.shortcuts import render

def display(request):
    return render(request, 'ex01/display.html')  # Display static HTML

4. Template Rendering

The render() function combines the template file (in this case, display.html) with the context (if any). The template file is an HTML file located in the templates folder.

Since this is a static page, the template might not contain any dynamic content. It simply loads the HTML and applies any static content like text, images, or CSS styling.

5. Response to the Client

After rendering the template, Django generates an HTTP response and sends the HTML back to the client's browser.

The browser then interprets the HTML and displays the static web page to the user.

Example Flow:

  • Request: The client requests http://localhost:8000/ex01/display.
  • Routing: The URL pattern in urls.py directs this request to the display view.
  • View: The display view loads the display.html template.
  • Template: The display.html template is rendered and returned as an HTML response.
  • Response: The browser displays the static page.
