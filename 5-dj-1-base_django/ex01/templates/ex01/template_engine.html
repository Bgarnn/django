1. Django's Default Template Engine

Django uses a powerful templating engine to dynamically generate HTML web pages. Templates can contain both static HTML and Django template tags (special placeholders) for dynamic content. This allows rendering data passed from the view into the template while following a clear separation of logic (in the views) and presentation (in the templates).

2. Blocks

Blocks are a key feature of Django's template inheritance system. A block tag defines a section of a template that can be overridden by child templates. It is often used in a base template, allowing other templates to extend and modify specific parts while reusing the overall structure.

{% block content %}This is the default content{% endblock %}

In a child template, you can override the block like this:

{% block content %}This is overridden content{% endblock %}

3. Loops (for ... in)

The for tag in Django is used to iterate over lists, dictionaries, or other iterable objects. It allows you to dynamically generate content for each item in a list or dictionary passed as context from the view.

  • {% for item in items %}
    • {{ item }}
  • {% endfor %}

This will loop through the items list and display each item in an unordered list.

4. if Control Structures

The if tag is used for conditional rendering in templates. It checks whether a condition is true and only displays the content within the block if the condition is met.

{% if user.is_authenticated %} Welcome back, {{ user.username }}! {% else %} Please log in. {% endif %}

This code checks if a user is authenticated and displays a personalized greeting or a login prompt accordingly.

5. Displaying Context Variables

Context variables are the data passed from the view to the template. In the template, these variables can be rendered using double curly braces {{ }}. Django replaces these variables with the actual values at runtime.

Hello, {{ user.username }}! Your favorite color is {{ favorite_color }}.

In this example, user.username and favorite_color are variables passed from the view and are displayed within the HTML content.

Example View Passing Context:

def example_view(request): context = { 'user': request.user, 'favorite_color': 'blue', } return render(request, 'example_template.html', context)
